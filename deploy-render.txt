DEPLOY KE RENDER (VulnVault)
============================

Prasyarat
  - Akun GitHub + akun Render (gratis): https://render.com
  - Repo ini sudah di-push ke GitHub (isi .env JANGAN ikut commit).

Langkah singkat
  1) GitHub: buat repo baru, push folder project ini (pastikan .env tidak ter-upload).

  2) Render → Dashboard → New → Blueprint.
     Pilih repo GitHub yang berisi project ini.
     Render akan membaca file render.yaml.

  3) Environment (WAJIB dicek setelah blueprint dibuat):
     Dashboard → service "vulnvault" → Environment:
       - APP_LOGIN_USER     = mis. admin
       - APP_LOGIN_PASSWORD = password kuat pilihan kamu
       - SESSION_SECRET     = biasanya sudah terisi otomatis (generateValue)
       - NODE_ENV           = production (sudah dari yaml)
       - FORCE_SECURE_COOKIE= 1 (sudah dari yaml)

  4) Deploy. Tunggu build "Live". Buka URL yang diberikan Render (*.onrender.com).

  5) Login dengan APP_LOGIN_USER / APP_LOGIN_PASSWORD yang kamu set.

Catatan free tier Render
  - Service bisa "sleep" jika lama tidak dipakai; akses pertama bisa lambat.
  - Disk bisa di-reset saat deploy ulang: data SQLite + upload bisa hilang.
    Untuk produksi serius nanti pertimbangkan DB + file storage tetap.

Update fitur ke internet
  - Ubah kode lokal → git commit → git push → Render auto-deploy (jika auto-deploy aktif).
